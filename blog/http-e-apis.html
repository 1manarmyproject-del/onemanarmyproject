HTTP e APIs — a língua que a internet fala

API é o cardápio de um restaurante. Você não entra na cozinha — pede pelo cardápio e a cozinha entrega. Entenda HTTP, REST, JSON e como integrações funcionam nos bastidores.
Fabio Xavier
Founder · OMA Project  ·  Co-criado com OMA

O cardápio do restaurante

Quando você vai a um restaurante, não entra na cozinha para preparar sua comida. Você consulta o cardápio, faz o pedido para o garçom, e a cozinha prepara e entrega. Você não precisa saber como o prato é feito — só o que está disponível e como pedir.

API (Application Programming Interface) funciona exatamente assim. É o cardápio de um serviço digital: define o que está disponível, como pedir, e o que será entregue. Você não precisa saber como o serviço funciona por dentro.

Exemplos do dia a dia

iFood mostra o mapa do entregador via API do Google Maps. Seu app de finanças puxa o saldo via API do banco. O e-commerce confirma o CEP via API dos Correios. Tudo isso acontece nos bastidores sem você ver.

O que é HTTP

HTTP (HyperText Transfer Protocol) é o protocolo de comunicação da web — a língua que navegadores e servidores usam para conversar.

Quando você abre um site, seu navegador envia uma requisição HTTP: "me dê o conteúdo de onemanarmyproject.com.br". O servidor responde com o HTML, CSS e JavaScript da página.

Os tipos principais de requisição:

  • GET — buscar informação. "Me dê a lista de produtos."
  • POST — enviar informação. "Aqui estão os dados do novo pedido."
  • PUT/PATCH — atualizar. "Mude o status desse pedido para entregue."
  • DELETE — remover. "Cancele esse pedido."

Cada resposta tem um código de status: 200 = ok, 404 = não encontrado, 401 = sem autorização, 500 = erro interno no servidor.

REST e JSON — o padrão moderno

REST é uma convenção de como organizar APIs — como estruturar as rotas, como usar os métodos HTTP, como retornar erros. A maioria das APIs modernas segue REST.

JSON (JavaScript Object Notation) é o formato de dados usado na comunicação. É texto estruturado que tanto humanos quanto máquinas conseguem ler:

{"nome": "Fabio", "empresa": "OMA", "plano": "pro"}

Todo sistema que você vai construir vai consumir e produzir JSON. É o idioma universal das integrações digitais.

Por que APIs importam para founders de PME

Cada integração do seu produto com o mundo externo é uma API. Pagamento (Stripe, Mercado Pago), email (SendGrid), WhatsApp (Evolution API), mapas (Google Maps), autenticação (Google OAuth) — tudo API.

Quando o time técnico fala "vamos integrar com X via API", eles estão dizendo: "vamos fazer nosso sistema conversar com o sistema de X usando o cardápio que X disponibilizou".

"Entender APIs é entender como o mundo digital se conecta. Cada produto moderno é uma orquestração de dezenas de APIs trabalhando juntas."
