DNS e Domínios — a lista telefônica da internet

DNS é a lista telefônica da internet. Você conhece o nome (google.com), o DNS traduz para o IP real. Entenda domínios, subdomínios e Cloudflare.
Fabio Xavier
Founder · OMA Project  ·  Co-criado com OMA

A lista telefônica que funciona em 50ms

Quando você digita google.com, seu computador não sabe onde esse endereço fica fisicamente. Ele pergunta a um servidor DNS: "qual é o IP desse domínio?" O DNS responde com um número como 142.250.78.46. Aí o navegador vai até esse endereço buscar o site. Tudo em menos de 50 milissegundos, completamente invisível.

A analogia

DNS = lista telefônica da internet. Você sabe o nome (domínio), o DNS fornece o número (IP). Sem DNS, você precisaria memorizar IPs para acessar qualquer site.

Domínio, subdomínio e IP

Domínio é o endereço amigável: onemanarmyproject.com.br. IP é o endereço real do servidor: 104.21.45.78. DNS faz a tradução entre os dois.

Subdomínios são departamentos do mesmo domínio: api.empresa.com.br, app.empresa.com.br, blog.empresa.com.br. Cada subdomínio pode apontar para um servidor diferente — seu site no Vercel, sua API no Hetzner, seu blog no WordPress — tudo sob o mesmo domínio principal.

Cloudflare — a guarita digital

Cloudflare fica entre o usuário e o seu servidor. Todo visitante passa por ela antes de chegar à infraestrutura real. O que ela faz: filtra ataques (DDoS), acelera carregamento com cache global, esconde o IP real do seu servidor, e oferece HTTPS gratuito.

A maioria dos produtos digitais sérios usa Cloudflare — não por luxo, mas por segurança e performance básicas.

TTL — quanto tempo o DNS demora para propagar

Quando você muda onde um domínio aponta (troca de servidor, por exemplo), essa mudança não é instantânea. O TTL (Time to Live) define por quanto tempo os servidores DNS do mundo inteiro guardam a resposta em cache. TTL de 3600 = propagação em até 1 hora. TTL de 86400 = até 24 horas.

Founder que não sabe disso fica confuso quando muda o servidor e o site "demora para atualizar". É o TTL fazendo seu trabalho.

"DNS é infraestrutura invisível — você só pensa nele quando algo dá errado. Entender o básico evita horas de debug desnecessário."
